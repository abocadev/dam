	- Ejecutable --> Fichero que contiene el codigo binario para ser interpretado en un PC.
	
	- Proceso --> Cada vez que una aplicacion se ejecuta, se genera un proceso de ejecucion.

DEFINICIONES: 

	MULTIPROCESO: Ejecucion de mas de un proceso a la vez. [Es casi mentira]
	
		Ventajas: 
			
			- Varios procesos en ejecucion.
			
			- Permite servicio a varios usuarios.
			
			- Aumenta el uso de la cpu.
	
	MULTIPROGRAMACION: Viene a ser lo mismo.
	
		Tipos:
		
			- Multitarea nula --> MS-DOS --> 1 proceso a la vez. [Obsoleto]
			
			- Multitarea corporativa --> Procesos de usuario ceden la CPU. --> Sistemas operativos anteriores a 1995. [Obsoleto]
			
			- Multitarea preferente --> El sistema operativo se encarga de decidir que proceso se ejecuta.
			
			- Multitarea real --> Multiples procesadores.
			
			

GESTION DE PROCESOS: --> Multitarea

	Es el S.O quien decide que se ejecuta:
	
		Gestion de procesos por lotes --> Conjunto de tareas cuyo resultados es lo unico que interesa, grupo de procesos muy similares.
	
		Gestion de procesos por interaccion --> Depende de la interpretacion del usuario.
	
		Gestion de procesos por tiempo real --> Esta todo el rato en ejecucion.

PROGRAMACION MULTIHILO --> Programacion que ejecuta distintos subprocesos.

	Tipos de programacion multihilo:
	
		- Programa de flujo unico --> Solo un proceso por programa = Ejemplos: Calculadora, MS-DOS/Terminal
		
		- [La programacion multihilo consiste en esto] Programa de flujo multiple --> Mas de un proceso por programa --> Ejecucion "a la vez" de una o mas tareas del mismo programa = Ejemplos: Firefox
		
	Hilo (subproceso) --> Es un flujo de control secuencial "independiente" [no hay relacion entre los distintos hilos] dentro de un proceso y esta asociado a una secuencia de instrucciones (codigo).
		|
		|--> Cada vez que se ejecuta un programa aparece 1 hilo.
		|
		|--> Unico de cada hilo:
		|
		|	- Tiene un pID (identificador).
		|	
		|	- Contador de programa (Para indicar a quien pertenece)
		|	
		|	- Registros
		|	
		|	- Pila (variables locales) 
		|
		|--> Compartido:
		
			- Codigo
			
			- Datos (variables locales)
			
			- Ficheros abiertos [Elemento peligroso | Zona critica (Synchronize)]
			
			
	Procesos de un hilo:
			
		1-. Un hilo no puede existir sin un proceso.
		
		2-. Un hilo no se puede ejecutar por si solo.
		
		3-. Dentro de lado proceso hay 1 o mas hilos.
		
		

VENTAJAS DE USAR USAR HILOS/SUBPROCESOS:

	- Se consumen menos recursos.
	
	- Se tarda menos tiempo en ejecutar.
	
	- Ideal para programas con dispositivos de E/S (Entrada y Salida)
	
	- Procesos que deben de ejecutar muchas tareas a la vez.
	
	- PRIORIDADES
